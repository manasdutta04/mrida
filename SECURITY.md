# Security Policy

## Supported Versions

MRIDA is pre-1.0 and under active development. Security fixes are applied to `main`.

## Reporting a Vulnerability

Please do not open public issues for sensitive vulnerabilities.

Report privately with:
- vulnerability summary
- impact assessment
- reproduction steps
- affected files/components
- suggested mitigation (if any)

Until a dedicated security email is published, open a private maintainer contact request in `SUPPORT.md`.

## Disclosure Process

1. Maintainers acknowledge report receipt.
2. Triage severity and scope.
3. Prepare and test fix.
4. Release patch and publish advisory when appropriate.
